Some hidden content here. To make an element initially hidden, add the .hidden class to it, and assign it an id. For example, this paragraph has class="hidden" id="hidden_text_demo". Comparison Blocks

These are great when you need content side by side, or side by side by side :)

When viewed on a smaller device, these comparison blocks will switch to a single-column so it's easy to read. This behavior also occurs when using internet explorer, or on other browsers which do not support CSS Grid.

3-item comparison block

Use the .block-comparison-3 class in a div or other block-level HTML element.

Item 1

The first item in your comparison block should have the .comp1 class.

This item also has the .lined class applied to it, giving it a clean border. This border's color is determined by your "line color" customization.

Item 2

The second item in your comparison block should have the .comp2 class.

If you need something that pops out a little bit more, the .shadowed class might be a good fit for you!

Item 3

The third item in your comparison block should have the .comp3 class.

I heard that rounded corners are all the rage now. Use the .rounded class to give your block-level element rounded corners.

2-item comparison block

Use the .block-comparison-2 class in a div or other block-level HTML element.

Item 1

You can combine class names to get their combined effects. For example, this particular div has class="comp1 lined rounded shadowed".

Item 2

Or you can always keep it simple, .comp1 to .comp3 classes work without any additional styling.

Other block-level elements

Plain Block

Block-level elements are very useful when you need to direct attention to certain areas in your webpage. Afterall, a page make up of a block of text can be very offputting.

These blocks create visual distinction and manageable chunks for the reader, making it more visually appealing and more readable. To create a block, add the .block class to a block-level HTML element.

You can also elevate paragraphs by adding a .border-left-accent class.

Sections With Sidebars

Some pages could really benefit from additional information or navigation in a sidebar. To create a section like that, use the .main-aside or .aside-main class in the containing element. The former has the main content on the left and the side content on the right; the latter is reversed.

Your main content should be enclosed by the <main> tag, and side content by the <aside> tag.

Here's a code example:

<div class="main-aside">
  <main>
    Main content
  </main>
  <aside>
    Sidebar content
  </aside>
</div>

The order you put the <main> and <aside> tags does not matter, and the containing element can be any block-level HTML tag, such as <section>, <div>, and <main>.

<div class="main-aside">
  <aside>
    Main content
  </aside>
  <main>
    Sidebar content
  </main>
</div>

Conside using wider .confined max-width values for these sections.

Confined sections

All the sections you've seen so far have the .confined class. This confines the element to a maximum width, so text don't stretch from edge to edge. This means the reader's eyes don't have to scan from the extreme left to extreme right, providing a more comfortable reading experience.

You can set the max-width for .confined elements when you customize Unilay.

However, if you want to do something clever with the design and would prefer take advantage of the entire width of the window, simply omit the .confined class. The homepage uses an unconfined block for the intro.

Note that all <section> elements have built-in margins - 4ems on top and 1em below.
